# Week 9: Design Patterns and Open Source Exploration
## Tasks Completed

During Week 9, I spent most of my time trying to understand what design patterns actually are and why they are important. At first, I didn’t really understand the concept fully, so I went through the course material slowly and tried to connect it with real examples. From the lecture notes, I learned that design patterns are basically reusable solutions to common coding problems, which helps developers write code that is easier to manage and understand (Blacquiere, 2026).

To understand this better, I researched a few common design patterns such as Singleton, Observer, Strategy, and Factory. I didn’t go too deep into the technical side, but I tried to understand them in a simple way. For example, Singleton makes sure only one instance of something exists, which I think is useful for things like app settings. Observer felt more interesting to me because it connects different parts of a system, like notifications updating automatically. Strategy and Factory were a bit harder to understand at first, especially Factory, but after reading examples, I started to see how they help organize code better.

I also completed the reading on how to contribute to open source, which honestly changed how I think about contributions. Before this, I thought contributing meant writing complex code, but now I understand that even small things like fixing documentation or reporting issues count as contributions.

After that, I explored GitHub more seriously. I looked at trending repositories and also checked beginner-friendly platforms like “good first issue.” I tried to find projects related to Python. Some repositories looked interesting, but I also noticed that many of them are quite complex, which made me realize I need to start small.

I also analyzed a few repositories by checking their issues, contribution guidelines, and activity. This helped me understand how to judge whether a project is suitable for beginners or not.

## What I Learned

This week helped me understand that programming is not just about writing code that works, but also about writing code that is organized and maintainable. Design patterns provide a structured way to solve problems, which saves time and effort in the long run.

I also realized that different design patterns are useful in different situations. There is no single “best” pattern, and choosing the right one depends on the problem. This made me think more carefully about how developers make decisions when building software.

Another important thing I learned was about open-source contribution. I used to think it was something only experienced developers do, but now I see that beginners can also participate. Understanding how to read documentation and explore repositories is also a skill that I need to improve.

## Reflection

In my opinion, this week felt a bit overwhelming at first. Compared to Week 8, the concepts were more technical and harder to understand quickly. Especially with design patterns, I initially felt like I was just memorizing names without really understanding them.

However, when I started thinking about real-life examples, things became clearer. For example, the Observer pattern made more sense when I thought about how apps send notifications. That moment actually helped me feel more confident because I could connect theory with something practical.

One thing I noticed about myself this week is that I tend to understand things better when I relate them to real-world situations instead of just reading definitions. This is something I want to keep doing in future weeks.

Exploring GitHub was also a new experience for me. I found it interesting but also a bit intimidating because of how advanced some projects are. At the same time, finding beginner-friendly issues gave me some confidence that I can start somewhere.

Overall, this week helped me realize that learning programming is a gradual process. I don’t need to understand everything perfectly right away. Instead, I need to keep exploring, practicing, and building my understanding step by step.
