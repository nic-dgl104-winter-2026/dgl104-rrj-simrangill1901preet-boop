## Tasks Completed

During Week 8, I started by reviewing the course materials to understand the transition into the second half of the semester. Since I was not present in class and the lecture video was unavailable, I relied on the written content provided on Brightspace to understand the key concepts such as user stories and functional user requirements. This required me to spend extra time going through the material carefully and making sure I understood the ideas properly.

As part of the required activities, I researched a new programming language. I chose Haskell because it was completely new to me. From my research, I learned that Haskell is a functional programming language that focuses on writing programs using mathematical functions and avoiding changes in data (HaskellWiki, n.d.). It is often used in research and by organizations that require high reliability, such as financial systems (Wikipedia, n.d.). I also explored beginner-friendly resources to get a basic understanding of how the language works and why it is different from more common languages.

I then worked on writing user stories using an app that I use regularly, which is Spotify. For example, one of my user stories was:
“As a user, I want to play songs from my playlist so that I can listen to music easily.”
I also created more specific versions of this story, such as playing songs offline or skipping tracks. Additionally, I thought about acceptance criteria, such as ensuring the play button works instantly and songs load without errors. This helped me understand how developers move from general ideas to specific technical expectations.

Another activity I completed was exploring GitHub. I went through the Explore and Trending sections and starred a few repositories that were beginner-friendly, especially those related to Python. This gave me a better idea of how developers share projects and collaborate in real-world environments.

Finally, I began thinking about my coding project for the semester. I also began thinking about my coding project and started planning how I would structure it using basic web development tools like HTML and CSS.

## What I Learned

This week helped me understand the importance of user-centred design in software development. Before this, I mostly thought about apps only from a user perspective, focusing on design and usability. However, learning about user stories made me realize that developers need to clearly understand user needs before starting to code.

I also learned that user stories are simple but powerful. Even though they are written in plain language, they play a major role in shaping how features are built. Functional user requirements take these ideas further by turning them into more structured and detailed specifications, which reduces confusion during development.

While researching Haskell, I was introduced to the concept of functional programming. Compared to other programming styles, it focuses more on functions and avoids modifying data directly. At first, this concept was confusing to me, but after reading more about it, I started to understand why it can make programs more predictable and easier to test (HaskellWiki, n.d.).

## Reflection

In my opinion, this week was both challenging and eye-opening. Since I missed the class and did not have access to the lecture video, I initially felt like I was falling behind. However, spending extra time reading the material actually helped me understand the concepts more deeply because I had to figure things out on my own.

One thing that stood out to me was how simple user stories look at first, but how important they are in real development. When I started writing multiple versions of the same user story, I realized how they can represent different user needs and situations. This made me think differently about how apps are designed.

I also found Haskell quite difficult compared to more common languages like Python. This made me realize that learning new programming languages can be challenging, especially when they follow completely different approaches. However, it also made me more interested in exploring different programming styles in the future.

Exploring GitHub was another interesting experience. I noticed that many projects are built by multiple contributors, which shows that software development is often a team effort. This was something I had not really thought about before.

Overall, this week changed my perspective on app development. Instead of thinking only about coding, I now understand the importance of planning, understanding user needs, and defining clear requirements before starting development. Moving forward, I want to improve my ability to write better user stories and become more comfortable exploring new programming languages.
