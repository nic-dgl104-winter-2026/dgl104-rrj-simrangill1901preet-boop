# Week 12: Functional Programming and Final Project Focus
## Tasks Completed

During Week 12, I focused on understanding the concept of functional programming by going through the lecture slides and course materials in detail.

From the materials, I learned that functional programming is based on evaluating functions and avoiding changes in data or state. I carefully reviewed the key principles of functional programming, including:

Pure functions
Immutability
No side effects
Function composition
Declarative programming style

I also spent time understanding how functional programming differs from other paradigms like object-oriented and procedural programming. One thing that stood out to me was how functional programming focuses more on describing what needs to be done rather than how to do it step by step.

A major part of this week was learning about functional-style APIs, especially methods like map, filter, and reduce. I reviewed multiple examples from the lecture slides that showed how these methods can replace traditional loops and make code more concise and readable. For example, instead of writing multiple lines of code using loops, functional programming allows chaining operations together in a more expressive way.

I also explored how these concepts apply across different programming languages such as JavaScript, Python, Kotlin, and Java. This helped me understand that functional programming is not limited to one language but is widely supported in modern development.

As part of the required activity, I focused on working on my coding project. I spent time planning how I can apply some functional programming concepts, such as using simpler functions and avoiding unnecessary complexity in my code. I also reviewed parts of my project to see where I could improve structure and readability.

In addition to this, I carefully read through the lecture slides to understand the advantages and disadvantages of functional programming. I noted that functional programming can improve readability and reduce bugs, but it can also be difficult to learn at first and may use more memory in some cases.

## What I Learned

This week helped me understand that functional programming is a different way of thinking about problem-solving in programming. Instead of focusing on step-by-step instructions, it focuses on transforming data using functions.

I learned that concepts like immutability and pure functions are very important because they make programs more predictable and easier to test. If a function always produces the same output for the same input, it becomes much easier to debug and maintain.

The idea of using map, filter, and reduce was one of the most useful things I learned this week. These methods allow developers to process data in a clean and efficient way without writing complex loops.

I also learned that functional programming is widely used in modern development, including web and mobile applications. Many popular tools and frameworks use functional concepts, which makes this knowledge very relevant for real-world applications.

## Reflection

In my opinion, this week was very interesting but also a bit challenging because functional programming requires a different mindset compared to what I am used to. At first, I found it slightly confusing to understand how everything works without traditional loops and step-by-step instructions.

However, after spending time going through examples, especially the ones using map, filter, and reduce, the concepts started to make more sense. I realized that functional programming can actually make code shorter and easier to read once you understand the logic behind it.

One thing I noticed is that I need more practice to fully feel comfortable with functional programming. It is not something that becomes clear immediately, but it improves with time and practice.

I also found it useful to compare functional programming with object-oriented programming from the previous week. This helped me understand that both approaches have their own strengths, and developers often use a combination of both depending on the situation.

Working on my coding project this week also made me think more carefully about how I write code. Instead of just focusing on making the program work, I started thinking about how to make it cleaner and more efficient.

Overall, this week helped me expand my understanding of programming beyond just basic concepts. It gave me a new perspective on how code can be written in a more structured and efficient way.
